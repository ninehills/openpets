# OpenPets v0.1 Smoke Test

Run this on a clean user or machine for each target platform before tagging `v0.1.0`.

## Desktop artifact

- [ ] Download the release artifact.
- [ ] Install or unzip the platform artifact.
- [ ] Launch OpenPets without a source checkout.
- [ ] Confirm the tray/menu-bar icon appears.
- [ ] Confirm the default pet appears.
- [ ] Confirm **Hide Pet** and **Show Pet** work from the tray menu.
- [ ] Confirm **Sleep** works.
- [ ] Confirm **Scale** menu updates the pet size.
- [ ] Confirm **Settings → Open Config File** works.
- [ ] Confirm **Settings → Reveal Config Folder** works.
- [ ] Quit and relaunch; confirm config persists.

## Platform-specific packaging

### macOS

- [ ] `.dmg` opens.
- [ ] `.zip` extracts.
- [ ] Gatekeeper behavior matches `INSTALL.md` if unsigned.

### Windows

- [ ] NSIS installer runs.
- [ ] Portable `.exe` launches.
- [ ] SmartScreen behavior matches `INSTALL.md` if unsigned.
- [ ] MCP `openpets_start` finds installed app from `%LOCALAPPDATA%\Programs\OpenPets` or documented override.

### Linux

- [ ] AppImage launches after executable bit is set.
- [ ] `.deb` installs on supported distro.
- [ ] Tray behavior is verified or documented as limited for that desktop environment.
- [ ] MCP `openpets_start` finds `openpets` on PATH, `/opt/OpenPets/openpets`, or documented AppImage override.

## IPC/client health

- [ ] With the app running, `openpets_health` reports running.
- [ ] With the app quit, `openpets_start` launches it.
- [ ] With the app already running, `openpets_start` does not create a duplicate process.

## Claude MCP

- [ ] Configure Claude Code with the documented MCP command.
- [ ] Restart Claude Code.
- [ ] `openpets_health` succeeds.
- [ ] `openpets_start` succeeds.
- [ ] `openpets_say` shows safe speech on the pet.
- [ ] `openpets_set_state` changes pet state.
- [ ] `openpets_release` releases the current MCP lease without quitting another session's pet.

## Known limitation checks

- [ ] If unsigned, Gatekeeper warning behavior matches `INSTALL.md`.
- [ ] CLI is documented as developer-only.
- [ ] Windows/Linux preview caveats are documented.
